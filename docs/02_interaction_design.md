# 交互设计方案 — 有道词典 Flutter 复刻版

## 1. 全局交互约定

| 规则 | Android | Windows |
|------|---------|---------|
| 返回 | 系统返回键 / 左上角← | Alt+← / 左上角← |
| 右键菜单 | 长按弹出 BottomSheet | 右键弹出 ContextMenu |
| 滑动操作 | 左滑删除历史/收藏 | — (右键删除) |
| 键盘 | 软键盘自动弹出 | 物理键盘，Enter 触发搜索 |
| 焦点 | 触摸驱动 | 键盘 Tab 导航 |
| 滚动条 | 无（移动习惯） | 始终显示滚动条 |

## 2. 首页交互流

```
启动
  │
  ▼
首页（搜索框未聚焦）
  │ 点击搜索框
  ▼
搜索框聚焦
  ├─ 显示键盘候选词（Android IME）
  ├─ 实时过滤历史词（输入≥1字符）
  │
  │ 输入停顿 300ms 触发模糊补全
  ▼
补全下拉列表（本地词典前缀匹配）
  │ 选择补全词 或 Enter/搜索键
  ▼
词典详情页
  │
  │ 返回
  ▼
首页（保留输入内容）
```

### 搜索框状态机

```
空闲 ──点击──▶ 聚焦
              │
              ├── 输入中 ──清除键──▶ 聚焦（内容清空）
              │       │
              │       └── 补全展开
              │
              └── 失焦 ──▶ 空闲
```

## 3. 核心交互详细说明

### 3.1 搜索交互

**触发搜索的方式：**
1. 键盘搜索键（Android）/ Enter（Windows）
2. 点击补全列表中的词条
3. 点击历史词 Chip
4. 点击每日一词卡片

**实时补全规则：**
- 输入延迟：300ms 防抖
- 最多显示：8 条补全
- 优先级：精确匹配 > 历史搜索 > 词典前缀
- 补全条目显示：词 + 首条简短释义（灰色）

**搜索框清除按钮：**
- 有内容时显示 × 按钮
- 点击 × 清空内容并保持聚焦状态
- 不清除历史记录

### 3.2 词典详情页交互

**Tab 导航（词典/翻译/例句/详解）：**
- 手势：水平滑动切换（Android）
- 鼠标：点击 Tab（Windows，不支持滑动切换）
- Tab 指示器：底部红色下划线，滑动动画

**发音交互：**
```
点击 [🔊英] 按钮
  → 图标变为 loading spinner（旋转）
  → 播放 TTS 音频
  → 恢复正常图标

若 TTS 引擎未初始化：
  → 按钮置灰 + tooltip "发音暂不可用"

若 TTS 播放失败（运行时错误）：
  → 图标短暂变为 error_outline（红色，500ms）
  → Android：轻微震动反馈（HapticFeedback.lightImpact）
  → Windows：顶部提示条 "发音失败，请检查系统 TTS 设置"
  → 恢复正常图标（可重试）
```

**收藏交互：**
```
未收藏状态：点击 ★
  → scale 1.0 → 1.3（100ms easeOut）→ 1.0（100ms easeIn）
  → 图标变为实心金色 ★
  → SnackBar："已添加到收藏" + [撤销]（Android）
    / 顶部提示条（Windows）

已收藏状态：点击 ★
  → 弹出确认对话框："取消收藏该单词？"
  → 确认 → 图标变为空心 ☆
  → SnackBar："已取消收藏"（Android）/ 顶部提示条（Windows）
```

**长按例句（Android）/ 右键例句（Windows）：**
- 弹出菜单：复制英文 / 复制中文 / 复制全部

### 3.3 历史记录交互

**Android - 左滑删除：**
```
左滑 50% → 显示红色删除背景 + 垃圾桶图标
左滑 > 75% → 自动确认删除 + 动画收起
左滑 ≤ 50% → 弹回恢复
```

**Windows - 右键菜单：**
```
右键词条 → ContextMenu
  ├── 查看详情
  ├── 添加/取消收藏
  └── 删除记录
```

**批量删除（两端通用）：**
- 长按（Android）/ Ctrl+A（Windows）进入多选模式
- 选择后显示底部操作栏：已选 N 项 [全选] [删除]

### 3.4 收藏页交互

与历史记录页相同，额外支持：
- 按字母分组（默认）/ 按时间排序（切换）
- 搜索过滤（顶部搜索框）

### 3.5 设置页交互

**主题切换（Windows）：**
```
点击主题模式选项
  → 弹出选择器：浅色 / 深色 / 跟随系统
  → 立即生效，无需重启
  → 持久化到 SharedPreferences
```

**AI 模型管理：**
```
未安装状态：
  → 显示 [下载] 按钮
  → 点击 → 跳转模型下载页

已安装状态：
  → 显示模型名称 + 大小
  → [切换模型] / [删除模型] 选项

下载中状态：
  → 显示进度条 + 百分比
  → [暂停] / [取消] 按钮
  → 支持断点续传（暂停后可继续）
```

**从文件导入模型：**
```
点击 [从文件导入 GGUF 模型]
  → 打开系统文件选择器（.gguf 文件）
  → 验证文件格式
  → 复制到应用模型目录
  → 显示导入进度
  → 完成后自动激活
```

## 5. 键盘快捷键（Windows）

| 快捷键 | 功能 | 实现 Intent |
|--------|------|-------------|
| Ctrl+F / F3 | 聚焦搜索框 | `SearchIntent` |
| Enter | 执行搜索 | — (TextField onSubmitted) |
| Escape | 清空搜索框 / 关闭弹窗 | `ClearSearchIntent` |
| Alt+← | 返回上一页 | `NavigateBackIntent` |
| Ctrl+D | 收藏/取消收藏当前词 | `ToggleFavoriteIntent` |
| Ctrl+C | 复制选中内容 | — (系统默认) |
| F5 | 播放当前词发音（美式） | `PlayPronunciationIntent` |
| Shift+F5 | 播放当前词发音（英式） | `PlayPronunciationBritishIntent` |
| Tab/Shift+Tab | 切换 Tab 页 | `SwitchTabIntent` |
| Ctrl+, | 打开设置 | `OpenSettingsIntent` |

所有 Intent 均在 `KeyboardShortcutsWrapper` 中注册完整的 `Actions` 处理器。

## 6. 通知与反馈

### Toast / SnackBar（Android）

| 场景 | 内容 | 时长 |
|------|------|------|
| 添加收藏 | "已添加到收藏" + [撤销] | 3s |
| 取消收藏 | "已取消收藏" | 2s |
| 复制成功 | "已复制到剪贴板" | 2s |
| 删除历史 | "已删除" + [撤销] | 3s |
| 发音失败 | "发音暂不可用" | 2s |
| 模型下载完成 | "AI 模型已就绪" | 3s |
| 模型下载失败 | "下载失败，请重试" + [重试] | 5s |

### Windows 顶部提示条

Windows 端使用应用内顶部提示条（高度 32px，背景 #1A1A1A，文字白色），替代 SnackBar，自动 3s 后消失。错误类提示背景使用 #B00020。

## 7. 加载状态设计

### 首次启动（词典数据库初始化）

```
全屏 Splash
  ┌─────────────────────────────┐
  │                             │
  │      [有道红 Logo]           │
  │      有道词典                │
  │                             │
  │  ████████░░░░  62%          │
  │  正在初始化词典数据...        │
  │                             │
  └─────────────────────────────┘
```

进度来源：词典 SQLite 数据库从 assets 复制到应用目录的字节写入进度。

### 查词加载

- 本地词典查询：无需 loading（< 50ms），直接展示结果
- 本地 LLM 解析（增强释义）：
  - 显示骨架屏（Shimmer 效果）
  - 超时 15s 后显示基础词典结果，并提示 "AI 增强超时，显示基础结果"
  - LLM 结果可在超时后追加（若最终完成）

### 模型下载加载

```
下载进度页
  ┌─────────────────────────────┐
  │  ████████████░░░░  68%      │
  │  550MB / 900MB              │
  │  预计剩余：2分30秒           │
  │  [暂停]  [取消]             │
  └─────────────────────────────┘
```

## 8. 错误处理交互

| 错误类型 | 展示方式 | 用户操作 |
|----------|----------|----------|
| 词典未找到 | 全页空状态 + 相似词建议 | 点击相似词跳转 |
| TTS 引擎未安装 | 发音按钮置灰 + tooltip | Android：引导安装 Google TTS |
| TTS 播放失败 | 图标短暂变红 + 提示 | 可重试 |
| 数据库初始化失败 | 红色提示条 | [重试] 按钮 |
| 数据库文件损坏 | Dialog 提示 | [重新初始化] 清除并重新复制 |
| 存储空间不足 | Dialog 提示 | 引导清理存储 |
| 模型下载失败 | 提示条 + [重试] | 支持断点续传重试 |
| LLM 推理超时 | 内联提示 | 显示基础结果，可手动重试 AI |
| LLM 加载失败 | 设置页错误状态 | [重新加载] / [重新下载] |

## 9. 无障碍设计

- 所有交互元素设置 `Semantics` label（包括发音按钮的当前状态：播放中/停止）
- 颜色对比度：正文文字满足 WCAG AA（4.5:1），大文字满足 AA（3:1）
- 字号支持系统字体大小缩放（0.85x ~ 1.3x），超出范围时截断而非溢出
- 发音功能为视障用户首选操作，Tab 键可直接聚焦
- Windows：支持键盘完整导航，焦点环可见（`FocusTraversalGroup` 合理分组）
- 加载状态通过 `Semantics(liveRegion: true)` 通知屏幕阅读器
